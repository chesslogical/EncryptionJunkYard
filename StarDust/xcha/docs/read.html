xcha

Fast in-memory XChaCha20-Poly1305 file encryption/decryption
Small CLI tool that encrypts or decrypts files in place using a fixed 32-byte key file (key.key).

Features

  • Uses XChaCha20-Poly1305 (256-bit key, 192-bit nonce)
  • Atomic in-place overwrite (via temporary file + rename)
  • Automatic encrypt/decrypt detection via magic header
  • Very fast – no password hashing / key derivation (fixed key)
  • 32 GiB file size safety limit
  • Single-file binary (after cargo build --release)

Security warning

This tool uses a static key file (key.key) stored next to the executable.

Anyone who gets access to both the program and key.key can decrypt all your files.
Use only on trusted / encrypted devices (full-disk encryption recommended).
Do not use for highly sensitive long-term secrets without additional protection.

Installation / Build

  1. Make sure you have Rust installed
  2. Clone or download the project
  3. Build release binary: 
    cargo build --release
  4. The binary will be in target/release/xcha.exe (Windows) or target/release/xcha (Linux/macOS)

Usage

Place a 32-byte random key file named key.key in the same directory as the executable.

Generate key (one time only):
openssl rand -out key.key 32
or (PowerShell):
$bytes = New-Object Byte[] 32; (New-Object Security.Cryptography.RNGCryptoServiceProvider).GetBytes($bytes); [IO.File]::WriteAllBytes("key.key", $bytes)

Basic usage – toggle encrypt/decrypt automatically:

xcha path/to/yourfile.txt

Force encryption (even if file already has magic header):

xcha --force-encrypt secret.pdf

Force decryption (even if no magic header):

xcha --force-decrypt document.docx

File format

OffsetLengthContent
014Magic header: XCHACHA_ENC_v1
1424Random XChaCha20 nonce
38variableXChaCha20-Poly1305 ciphertext + 16-byte authentication tag
